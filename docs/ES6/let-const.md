# let-const
为什么要使用这两哥们，那肯定是var不行啊，要是var行的话，那就不需要这两个哥们了，var怎么就不行了呢

## var的特点
  1. var声明的变量可以多次声明
  2. 无法控制修改，不同的类型居然也可以修改赋值
  3. 作用域是函数作用域（联想闭包 在没有块级作用域的时候的一种解决方案）
  4. 变量提升（这个应该还不算缺点）

  ```js
  var a = 5
  a = [1,2,3,4]
  var a = {a:5,b:'asfd'}
  ```
  完全没毛病，这显然是不好的

## let/const
对比var的问题可想而知这两个的好处
1. let/const声明的变量不可多次声明
2. 控制修改，const不允许修改类型（通常会把他当作一个常量来使用，但是啊，他的值其实是可以修改）
3. 作用域是块级作用域
